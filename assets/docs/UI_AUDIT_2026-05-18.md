# Frontend UI/UX Audit

Last updated: 2026-05-18

## Scope and discovery

- Frontend stack: Angular standalone components with SCSS.
- Routes: `/`, `/clinical-sessions`, `/data`, `/model-config`, `/sessions/:sessionId/timetable`.
- Global tokens and common primitives live in `app/client/src/styles.scss`.
- Reusable UI lives under `app/client/src/app/components`; route screens live under `app/client/src/app/pages`.
- The project already has a documented design system in `assets/docs/UI_STANDARDS.md`; the main issue is inconsistent adoption rather than absence of standards.

## Executive summary

The product is visually coherent overall, but several screens have drifted away from the shared system:

1. **Clinical Sessions** uses many local hex values, 32–34px controls, and bespoke radii/spacing rather than shared tokens.
2. **Patient Timetable** is effectively a second mini design system with its own palette, spacing, and control styles.
3. **Interactive states** are inconsistent: many controls rely on hover/active styling but do not expose a clear `:focus-visible` treatment.
4. **Responsive behavior** is present, but some dense toolbars and tab rows need safer wrapping/overflow behavior.
5. **Reduced motion support** is not expressed globally despite animated transitions elsewhere in the stylesheet.

No critical layout breakage was confirmed from code inspection alone. Visual breakpoints and real rendered contrast should be validated in-browser after implementation.

## Quick wins selected for implementation

1. Add a global interactive focus-visible baseline plus reduced-motion support. **Completed**
2. Normalize Clinical Sessions controls, spacing, radii, and tab overflow to the shared token system. **Completed**
3. Rebase Patient Timetable on the shared palette and component scale while preserving its information architecture. **Completed**
4. Tokenize the small Data Inspection RAG path editor rule set. **Completed**
5. Remove oversized empty-state card heights from the mobile Clinical Sessions layout. **Completed**

## Items needing visual verification

- Exact contrast ratios for page-specific gradients and pill states.
- Clinical Sessions tab behavior at narrow widths with long localized labels.
- Timetable lane readability on small laptop widths and mobile stacking.
- Keyboard traversal order and focus visibility for all tab-like controls, toolbar buttons, and timeline events.

## Remaining follow-up

- **Model Config stylesheet consolidation** remains the largest maintainability issue. The active rules are spread across a long sequence of historical overrides in `app/client/src/styles.scss`; consolidating them safely should be handled as a dedicated refactor with visual comparison before and after rather than folded into a token-normalization sweep.
- **Model Config stylesheet consolidation** has started: one superseded dark-theme override block was removed after confirming the later palette-repair block fully replaced it. The broader override stack is still a dedicated follow-up refactor because active layout rules remain interleaved across older and newer sections.

## Implementation intent

This pass intentionally avoids redesigning:

- the DILI workbench information architecture,
- the model configuration layout,
- the timetable lane model,
- or Clinical Sessions workflow semantics.

The goal is system compliance, not visual reinvention.
